## 1. 실험/데이터 맥락: CS Runs vs MDIS Runs

- **CS Runs (Charge State / Cross Section)**
  - 동위원소별(예: O18~O22, Ne24~Ne26)로 분리
  - 분석 목표: 안정적인 컷(특히 TSum)을 만들어 “동일 룰”로 적용 가능하게 하는 것
- **MDIS Runs (Momentum Distribution)**
  - setting(예: -12%, 0%, +7% 등) 변화에 따른 분포/효율/트리거 영향 확인
  - 분석 목표: setting 변화에도 **컷이 안정적으로 동작**하는지 검증

---

## 2. 데이터 흐름: 이벤트가 어떻게 “좋은 이벤트”가 되는가

권장 “기본 처리 파이프라인(초안)”:

1. **기본 트리거/품질 조건**: 유효한 엔트리(예: SSD entries) 등
2. **TSum 컷 (TXSum/TYSum)**: 좌/우 타이밍의 합이 물리적 범위인 이벤트만
3. **TDC/Timing 컷**: 오프타이밍/잘못된 트리거 조합 제거(필요한 채널만)
4. **기하학적/수용각 컷**: PPAC X/Y, SSD 위치 등을 이용한 acceptance 정의
5. **효율/비율 계산**: “컷 이후 hit 비율”로 PPAC 효율 산출

핵심 포인트:
- **TSum은 ‘조합(Combination)’ 문제를 줄이는 1차 방어선**
- **TDC 컷은 ‘타이밍/트리거 기원’ 문제를 다루는 2차 방어선**

---

## 3. PPAC 멀티힛/조합 문제: 왜 TSum이 필요한가

관측:
- TXSum 분포에 **정상 피크 외에 이상 피크(예: ~10000 부근)**가 나타남
- 이는 “서로 다른 레이어/히트의 Left, Right가 잘못 조합된 결과”일 가능성이 큼

실무적 결론:
- TXSum/TYSum에서 **앞쪽(작은 값) 영역을 잘라** 이상 조합을 줄이는 것이 합리적
- 단, 컷은 너무 공격적으로 잡으면 물리 이벤트도 제거할 수 있으므로 “검증 가능한 규칙”이 필요

---

## 4. TSum 컷 설계: ‘공통 룰’ 만들기

12/04 로그에서의 “공통 TSum 컷(예시)”:

- **F1 Up**
  - X: 0 ~ 4500
- **F2 Down**
  - X: 0 ~ 3500
  - Y: 400 ~ 4800
- **F3 Up**
  - X: 400 ~ 4000
  - Y: 400 ~ 4000
- **F3 Down**
  - X: 800 ~ 5000
  - Y: 1000 ~ 5000

해석:
- “모든 런에 동일 적용 가능한 넓은 컷”을 우선 제안
- 이후 setting/동위원소별로 “과학적 이유가 있을 때만” 수정

## 5. TDC 컷: 언제/어디에 필요한가

관측 (예: 12/04, 12/05, 12/09):
- **F2/F3는 TDC/TSum 컷으로 잃는 이벤트가 거의 없다**(컷이 비교적 안전)
- **F1은 컷에 민감**해서 효율(이벤트 보존율)이 크게 떨어질 수 있음

실무적 결론:
- F2/F3: **TDC 윈도우 컷을 기본으로 포함**해도 안전할 가능성이 높음
- F1: “왜 떨어지는지(트리거/멀티힛/조합)” 원인을 더 확인하고, **가능하면 느슨하게/조건부 적용**

---

## 6. CS vs MDIS 비교: 컷이 효율에 미치는 영향

예시 (12/09: CS Runs, 표 기반 요약):
- F2는 **~99% 수준**으로 매우 안정적
- F3 Up/Down은 “컷 적용으로 변화가 매우 작음(permille 수준)”
- F1은 컷 적용 시 손실이 비교적 큼(수 %~10% 수준 가능)

핵심 메시지:
- 분석 품질(가짜 이벤트 제거)을 위해 TSum/TDC는 효과적이지만,
  - **F1은 별도 전략(원인 규명 + 완화)**
  - **F2/F3는 기본 적용 후보**

## 7. 슬릿 오프셋 이슈: F3 Up Y가 왜 잘리는가

관측 (11/21, 12/04):
- F3 Up에서 “조합되지 않는 모든 데이터”까지 봐도 **Y 절단이 여전**
- 특정 timing/anode 조건에서 **F3 Up Y < -30** 영역이 비정상적으로 절단/왜곡

가설:
- 빔라인/슬릿 중심이 이동했거나, 하드웨어 세팅이 “원래 좌표계”와 어긋남

분석 대응:
- 단기: acceptance 정의 시 **Y 절단을 반영**하거나, 대칭 가정(예: +Y만 사용 후 대칭 추정)을 명시
- 중기: 슬릿/기계 오프셋을 확인해 **좌표계 보정(Offset correction)** 반영

## 10. Appendix: 용어 빠른 정의

- **PPAC**: 빔 위치/타이밍을 측정하는 검출기
- **TSum (TXSum/TYSum)**: 좌/우(혹은 상/하) 타이밍 신호의 합. “정상 조합”을 고르는 데 유용
- **TDC**: Time-to-Digital Converter. timing window 컷으로 오프타이밍 제거
- **PPT**: PPAC timing(로그에서 PPT로 표기). 트리거/윈도우 검토 대상
